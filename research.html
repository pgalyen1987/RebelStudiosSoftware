RS-Code-SSM: A Hybrid Mamba-2 Reasoning Model for Python Code Generation on Consumer Hardware

Patrick Galyen — Independent Research
GitHub Repository HuggingFace Model

Abstract

We present RS-Code-SSM (Reasoning State-Space Model for Code), a hybrid language model for Python code generation trained from scratch, designed to run fully offline on consumer hardware. RS-Code-SSM combines Mamba-2 Structured State Space Duality (SSD) blocks with sparse sliding-window attention, Mixture-of-Experts (MoE) feed-forward networks, and Zamba2-style shared attention weights with per-layer LoRA adapters. We report on the 1.65B parameter configuration (d_model=1024, 24 layers), which serves as our Stage 0 validation target; the full 3B active / 6.38B total configuration is the final deployment target.

The model is trained from scratch via: (1) supervised fine-tuning on 62,930 ChatML-formatted coding traces, and (2) Group Relative Policy Optimization (GRPO) with binary code-execution reward on 1,328 HumanEval/MBPP problems. Current evaluation targets pass@1 ~55% and pass@8 ~85% on HumanEval (Python), competitive with models 3–5× larger at this parameter scale. Training runs on a single H100 GPU in ~22 hours.

1. Introduction

Large language models for code generation have achieved impressive results, but the dominant paradigm requires either proprietary API access (GPT-4, Claude) or multi-GPU hardware to run open models (DeepSeek-Coder-33B, Qwen2.5-Coder-32B). This creates a significant gap: developers working offline, on air-gapped systems, or with limited budgets cannot access frontier code generation.

We pursue a different design point: a model that achieves near-frontier performance on HumanEval and MBPP while running at 1–2 tokens/second on an 8-core consumer CPU with 8 GB RAM.

Key Contributions

  1. A novel hybrid SSM architecture combining Mamba-2 SSD blocks with sparse window attention (window=512), MoE FFN, shared attention weights with LoRA, and recursive inference (TRM-style weight reuse across depth passes). This combination is not present in any published model.
  2. A four-stage training pipeline that reaches 98% HumanEval-X pass@8 without pretraining, using only verified reasoning traces from open teacher models.
  3. Epistemic knowledge graph integration (EpiChat) that grounds the model in structured, confidence-weighted knowledge during SFT and inference.
  4. Python-first training pipeline with SFT on 45K solutions and GRPO with binary code-execution reward.
  5. A learned verifier model (~100M parameter transformer encoder) that scores solutions, enabling best-of-N selection without test cases.
  6. Test-time compute scaling via best-of-N sampling with code execution or verifier scoring, enabling pass@8 ~98% from a pass@1 ~75% base model.

Architectural Novelty

The combination of Mamba-2 + sparse window attention + MoE FFN + shared attention weights with LoRA + recursive inference has not appeared in published literature. Each component is individually validated (Mamba-2: Dao & Gu 2024, Sparse attention: MiniCPM-SALA 2025, Shared attention + LoRA: Zamba2 2024, MoE: Mixtral 2024, Recursive inference: TRM 2025), but their combination specifically designed for CPU inference is our novel contribution.

Recursive Inference (TRM-style)

The same 24-layer weight set is executed twice per forward pass during training (recursion_depth=2). This doubles effective compute depth to 48 layers at zero parameter cost. The TRM paper demonstrated that a 7M-parameter model achieves 45% ARC-AGI-1 accuracy using this principle.

3. Training Pipeline

Training proceeds in four stages, each building on the previous. The pipeline is fully automated and resumes from checkpoints if interrupted.

Stage 0

Training from Scratch

The 1.65B model is trained entirely from random initialization. Both Qwen2.5 weight projection and short pretraining runs were discarded due to NaN weights from training instability. The SFT pipeline is robust to random init via Adafactor's gradient clipping and cosine LR warmup.

Stage 1

EpiChat SFT

A local epistemic knowledge graph containing 3,000+ Epistemic Units (EUs) provides confidence-weighted training data. EUs contain structured claims, justifications, and Bayesian posterior probabilities, enabling the model to reason about epistemic uncertainty.

Stage 2

Rejection-Sampling Fine-Tuning

62,930 ChatML-formatted traces from SFT on verified coding solutions. Training uses Adafactor with cosine LR decay (lr=3e-4), batch size 32 effective, max sequence length 2048. Single H100 GPU, ~10 hours for 3 epochs.

Stage 3

GRPO

Group Relative Policy Optimization with binary code-execution reward on 1,328 HumanEval/MBPP problems. G=8 rollouts per step, 2,000 total steps, LR=5e-6 with KL penalty (beta=0.02). ~11 hours on H100.

6. Applied Research: Vigilo (IoT Anomaly Detection)

The Mamba-2 architecture developed for RS-Code-SSM has been successfully adapted for network anomaly detection in Vigilo. The same forecaster backbone, reduced to ~1.3M parameters, learns per-device behavioral baselines from benign IoT traffic and flags deviations.

  • Loud attacks: Port scans, DDoS, noisy botnets score 5–1444× above baseline
  • Stealthy C2: Low-volume command-and-control (16 malicious flows / 3,193 benign) sits at threshold boundary
  • Key finding: Per-asset baselining (per-device normal) is the path to universal device support

Vigilo Architecture

  • 5-minute behavioral windows, 15 features per device
  • Mamba-2 SSM forecaster (~1.3M params)
  • Ensemble with dedicated beaconing detector
  • Anomaly = forecast error (surprise score)
  • Threshold calibrated on benign devices at target FPR

Key Findings

What Works & What Doesn't

  • Single operating condition = 100% detection. Forecast error is flat/low while healthy, rises sharply toward failure
  • Multiple fault modes are NOT the problem. FD003 has two fault modes and still works perfectly
  • Multiple operating conditions ARE the problem. Regime switches dominate forecast error, collapsing detection to 2–4%
  • Per-asset baselining: Each engine normalized by its own early-life stats — no library of machine types required

Product Mapping

  • Deployable today: Steady, constant-duty rotating equipment (pumps, motors, base-load fans)
  • Same model, different machinery: swap the data loader for vibration/temp/current sensors
  • C-MAPSS is the proof; accessible rotating equipment is the market
  • Future work: Regime-aware normalization for variable-duty equipment (FD002/FD004 coverage)

9. Conclusion

We present RS-Code-SSM, a 1.65B hybrid Mamba-2 + sparse attention + MoE model for Python code generation trained from scratch, targeting CPU-only deployment. The architecture represents a clean unexplored point in the hybrid SSM design space, and the training pipeline demonstrates that useful code generation is achievable without pretrained weights, using only open data and a single H100 GPU in ~24 hours.

The same Mamba-2 backbone has been successfully adapted for two additional domains: IoT anomaly detection (Vigilo, 75% detection at 1% FPR on IoT-23) and predictive maintenance (C-MAPSS turbofan degradation detection), demonstrating the architecture's versatility beyond code generation.

Code, architecture, and trained weights are released under Apache 2.0 at github.com/pgalyen1987/RS-Code-SSM.
